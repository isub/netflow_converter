netflow Converter поддерживает 9ю и 10ю версии NetFlow
параметры командной строки:
	converterVerbosity - уровень логгирования. Рекомендуемое значение '0'
	dataDir=<имя директории> - имя директории с исходными данными
	dataDirRecursive - если задан этот параметр конвертор будет осуществлять поиск файлов с исходными данными во вложенных директориях
	filterTimeFormat=<формат времени> - формат представления времени для фильтра. Используется для преобразования значений, заданных параметрами filterTimeStart и filterTimeStop
	filterTimeStart=<время начала потока> - время начала потока. Используется для выборки данных из файлов с исходными данными, а так же для отбора файлов с исходными данными
	filterTimeStop=<время окончания потока> - время окончания потока (используется по аналогии с параметром filterTimeStart)
	filterTimeToleranceBefore=<корректировочное значение> - временной интервал (в сек). Корректировочное значение вычитается из значения, полученного в параметре filterTimeStart. Используется только при отборе файлов с исходными данными
	filterTimeToleranceAfter=<корректировочное значение> - временной интервал (в сек). Корректировочное значение прибавляется к значению, полученному в параметре filterTimeStop. Используется только при отборе файлов с исходными данными
	filterAddrSrc=<ip-адрес> - ip-адрес источника. Используется фильтром для выборки записей
	filterAddrDst=<ip-адрес> - ip-адрес получателя. Используется фильтром для выборки записей
	outputFormatDate=<формат времени> - формат времени, используемый для вывода данных
	outputFormatDateAdd=<sec|msec|usec|nsec> - расширение формата вывода времени: sec - не изменяет формат, msec - дополняет миллисекундами, usec - микросекундами, nsec - наносекундами
	converterThreadCount=<количество потоков> - количество потоков обработки исходных данных. Не имеет смысла задавать значение равное или превышающее число процессоров в системе
	converterConf=<имя файла> - имя файла конфигурации конвертора
	converterResultFile=<имя файла> для вывода результатов
	help - вывод помощи и выход из программы
Все параметры командной строки поддерживаются в файле конфигурациии
кроме того в конфигурационном файле должны быть описаны следующие параметры:
converterOutputTemplate = { "<первое поле>"; "<второе поле>"; "<N-ное поле>"; }; - список имен атрибутов, которые следует выводить в качестве результата выборки
converterMapper = { <9|10>; "<исходное значение>"; "<новое значение>"; <делитель>; }; - параметр маппинга:
	первый параметр из списка, заключенного в фигурные скобки, - номер протокола netFlow;
	второй - имя атрибута согласно спецификации;
	третий - измененное значение имени атрибута для приведения в соответсвии с именами атрибутов в нотации шаблона вывода и фильтров
	четвертый - делитель (для секунд - 1, для миллисекунд - 1000 и т.д.)
Что надо знать для правильной настройки конвертора:
	шаблон вывода и фильтр работает с именами атрибутов, предварительно обработанных процедурой маппинга
	шаблон вывода поддерживает произвольные имена атрибутов:
	фильтр, напротив, работает с ограниченным перечнем унифицированных имен атрибутов:
		flowStart - время начала потока в секундах
		flowEnd - время окончания потока в секундах
		sourceIPv4Address - ipv4-адрес источника
		destinationIPv4Address - ipv4-адрес получателя
	например, в конфигурационном файле заданы следующие параметры:
		outputFormatDate = "%Y.%m.%d_%H:%M:%S";
		outputFormatDateAdd = "msec";
		converterOutputTemplate = { "Дядя Вася"; };
		converterMapper = { 10; "flowStartMilliseconds"; "Дядя Вася"; 1000; };
	процедура маппинга получив на входе значение flowStartMilliseconds=1550672116123
	преобразует его согласно шаблону outputFormatDate="%Y.%m.%d_%H:%M:%S" в "20.02.2019_14:15:16"
	затем, в соответсвии с параметром outputFormatDateAdd="msec" допишет к полученной строке ":123"
	в итоге получим "20.02.2019_14:15:16:123"
	имя атрибута, согласно параметру converterMapper = { 10; "flowStartMilliseconds"; "Дядя Вася"; 1000; };
	преобразуется из "flowStartMilliseconds" в "Дядя Вася", исходное значение 1550672116123 делится на 1000 и передается фильтру
	фильтр не поддерживает атрибут "Дядя Вася", поэтому фильтрация по времени не сработает и обработка записи будет продолжена
	и если запись будет соответствовать другим параметрам фильтра, то в результатах выборки мы увидим:
		Дядя Вася
		20.02.2019_14:15:16:123
	если же мы исправим параметр converterMapper следующим образом:
		converterMapper = { 10; "flowStartMilliseconds"; "flowStart"; 1000; };
	то фильтру передается значение flowStart=1550672116. Фильтр в этом случае корректно обработает запись по заданному времени.
	но теперь в результатах выборки это значение мы не обнаружим
	для этого надо исправить параметр converterOutputTemplate следующим образом:
		converterOutputTemplate = { "flowStart"; };
	в этом случае мы увидим в результатах выборки такие записи:
		flowStart
		20.02.2019_14:15:16:123
следующие параметры командной строки переопределяют значения, заданные в конфигурационном файле:
	- dataDir
	- dataDirRecursive
	- filterTimeFormat
	- filterTimeStart
	- filterTimeStop
	- filterTimeToleranceBefore
	- filterTimeToleranceAfter
	- filterAddrSrc
	- filterAddrDst
	- outputFormatDate
	- outputFormatDateAdd
	- converterThreadCount
	- converterResultFile
